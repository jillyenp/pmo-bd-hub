Build Your Own BD Hub Microsite

A step-by-step guide to building a fully functional, PwC-branded internal website using Claude AI — from materials prep to deployment.

Created by Jillyen Pagcaliwagan, PMO BD Lead • Based on the PH PMO BD Hub build experience

What You'll Build

A multi-page, responsive, PwC-branded microsite with these pages:

Home Page

Hero banner, service overview, approach, case studies, quick links

Guidebooks

Links to playbooks with summaries & open buttons

Templates Library

Searchable, filterable table with links to files

Service Catalogue

Detailed service descriptions, lifecycle, case studies

BD Tools

Pitches, FAQs, positioning guides, capability statements

Framework

Visual governance or delivery framework

Our Team

Team profiles with photos, roles, experience

Chatbot

Built-in assistant answering FAQs about your services

Gather these before your first message to Claude. The more you provide upfront, the better the output.

⚡ Biggest Lesson Learned: The Templates Excel with real SharePoint links is the single most important file. Get this right first. If links are placeholder text, they won't work on the live site and you'll need to update them manually later.

Step 7: Deploy on GitHub Pages

Once you're happy with local testing, deploy online so your entire team can access it via a shareable URL.

🚫 Don't use SharePoint Embed SharePoint strips out CSS and JavaScript when you embed HTML. Your beautiful site will look like plain text. Use GitHub Pages instead — it renders everything perfectly.
1

Create a GitHub Account

Go to github.com → Sign up (free). Use any email.

2

Create a New Repository

Click "+" → New Repository. Name it (e.g., pmo-bd-hub). Set to Public. Don't add a README.

💡 Why Public? GitHub Pages on free accounts requires Public repos. Nobody will find your site unless you share the link — it's like an unlisted YouTube video.
3

Upload Your Files

Click "Upload files" → use "choose your files" button (not drag-and-drop — PwC Forcepoint blocks folder drag). Upload in rounds:

Round 1: Upload all .html files + README.md
  (Use "choose your files" → select all → commit)

Round 2: Create css/styles.css
  ("Add file" → "Create new file" → type css/styles.css)
  (The "/" auto-creates the folder!)
  (Copy-paste contents from your local file → commit)

Round 3: Create each JS file the same way:
  js/main.js
  js/chatbot.js
  js/templates.js
  js/chatbot-kb.json

Round 4: Upload photos:
  Create photos/.gitkeep first (creates folder)
  Navigate into photos/ → Upload files → select photos
⚠️ Critical: index.html must be at root level Upload the contents of your folder, not the folder itself. If index.html ends up inside a subfolder, the site won't work.
4

Enable GitHub Pages

Go to Settings → Pages (left sidebar) → Branch: main → Folder: / (root) → Save.

5

Your Site is Live! 🎉

Wait 1-2 minutes, then refresh the Pages settings page. You'll see:

✅ Your site is live at:
https://yourusername.github.io/pmo-bd-hub/
6

Share With Your Team

Share the URL directly, or add as a Microsoft Teams tab for daily access:

Teams → Your channel → "+" (Add tab)
→ Select "Website"
→ Name: "PMO BD Hub"
→ Paste your GitHub Pages URL
→ Save ✅

Step 8: Update Your Live Site

Three ways to make changes after deployment. Changes go live in ~1 minute after each commit.

Edit on GitHub

Click any file → pencil icon ✏️ → make changes → "Commit changes". Live in ~1 minute.

Best for: small text/link fixes

Re-upload File

"Add file" → "Upload files" → select updated file → Commit. It replaces the old version automatically.

Best for: bigger file updates

Delete + Re-create

Click file → "..." → "Delete file" → Commit. Then upload or create the correct version.

Best for: fixing mistakes
💡 If changes don't appear: Hard refresh your browser with Ctrl + Shift + R. GitHub Pages can cache for a few minutes. Or open in an incognito / private window.

Pro Tips & Lessons Learned

Things we learned the hard way so you don't have to.

📋 Prep is everythingThe more materials you give Claude upfront, the fewer change rounds you'll need.
📊 Templates Excel is kingGet this perfect first — with real SharePoint links. Placeholder links = broken buttons.
📦 Bundle change requests5 changes in one message beats 5 separate messages. Saves time and keeps files in sync.
📤 Re-upload after local editsIf you edited files locally, upload them back before asking Claude for more changes.
📸 Name photos simplyUse Jill.png not "jill photo final (2).PNG". Crop to square. Any size works — CSS handles the rest.
🚫 Skip SharePoint embedIt strips all CSS/JS. Your site will look like plain text. Use GitHub Pages instead.
🔍 Test locally firstAlways test by double-clicking index.html on your computer before deploying to GitHub.
✏️ "PwC" not "PWC"Tell Claude explicitly if casing matters. Brand details get missed unless you specify.
🤖 Chatbot is fragileOne misplaced quote in chatbot.js breaks the entire chatbot. Always test it after changes.
🔗 Test all linksClick every button after deployment. Placeholder URLs from the build will need replacing.
📱 Check mobile viewPress F12 → toggle device toolbar to preview phone view. The site is fully responsive.
🔄 You can always come backUpload files to a new Claude chat anytime for more changes. Just provide context on what you need.
