Step 1: Prepare Your Materials

Gather these before your first message to Claude. The more you provide upfront, the better the output.

# Material Format Priority What Claude uses it for & tips
1 Service Catalogue / One-Pager PDF, PPTX Must have Drives home page, service catalogue, and BD tools content. Tip: The FY26 one-pager worked great for us.
2 Guidebooks / Playbooks PPTX + links Must have Creates the guidebooks page with summaries. Tip: Provide the SharePoint links — Claude will add "Open Presentation" buttons.
3 Templates List Excel (.xlsx) Must have Builds the searchable template library. Tip: Include columns for name, category, file type, description, and SharePoint link. This is the most important file.
4 Team CVs PPTX, DOCX Must have Creates team page with profiles. Tip: Claude will extract names, roles, years of experience, and certifications.
5 Team Photos PNG, JPG Nice to have Profile photos on team page. Tip: Name each file by person (e.g., Jill.png). Square crop works best. Can be added later.
6 Governance / Framework Visual HTML, PPTX, IMG Nice to have Becomes its own Framework page. Tip: If you have it in HTML already, Claude can integrate it directly.
7 Case Study Details Any Nice to have Adds credibility to service catalogue. Tip: If not provided, Claude generates realistic examples based on your services.
⚡ Biggest Lesson Learned: The Templates Excel with real SharePoint links is the single most important file. Get this right first. If links are placeholder text, they won't work on the live site and you'll need to update them manually later.

Step 4: Request Changes

You will want changes — that's normal! Here's how to request them effectively.

✅ Golden Rule: Bundle your changes! Don't send one change at a time. Collect all issues, then send them in one message as a bulleted list. This saves time and prevents files from getting out of sync.
Fix styling "The font color in service catalogue is not readable — please fix"
Remove something "Please remove the BD sponsor section from the home page"
Rename items "Rename 'Playbook' to 'Guidebook' everywhere for consistency"
Add a new page "Can you add a Framework page? I've attached the file"
Fix broken links "The link to VRO Guidebook is not working"
Add photos "I've attached team photos. Please add them to the team page"
⚠️ Important: If you edited files locally, re-upload them! Say: "I made some changes. Here are my latest files. Please use these as the base and apply these changes: [list]" — this prevents Claude from overwriting your edits.

For small changes, you can edit the files yourself — no need to go back to Claude.

Tip: If extension changes, open team.html → Ctrl+H →
Find: .png" → Replace: .jpg" → Replace All

Add a New Template

Open js/templates.js, find the templateData array, and add a new entry:

{
  name: "Your Template Name",
  category: "PMO",
  fileType: "xlsx",
  description: "What it does",
  link: "https://sharepoint-link"
},

Update Guidebook Links

Open guidebooks.html and use Find & Replace:

Ctrl+H
Find: https://old-placeholder-url
Replace: https://real-sharepoint-link
→ Replace All → Save

Update Chatbot Answers

Open js/chatbot.js, find the knowledgeBase array, edit or add:

{
  keywords: ['your', 'keywords'],
  answer: 'Your answer here'
},

⚠️ Use double quotes inside answers
for links: <a href="page.html">

Step 6: Deploy Your Site

Get your site live and shareable.

Option A: Local (Testing)

Just double-click index.html — it opens in your browser with full styling. Share the folder via Teams/OneDrive.

Pros: Instant, no setup
Cons: Only you can see it
Best for: Testing & reviewing

Option B: GitHub Pages (Recommended)

Free hosting with a shareable URL. Full CSS/JS support.

1. Create account at github.com
2. New repo → name it → Private
3. Upload all files (contents, not folder)
4. Settings → Pages → main branch → Save
5. Wait 2 min → live URL!

⚠️ Don't use SharePoint embed — it
strips all CSS and JS. Site looks basic.

Pro Tips & Lessons Learned

Things we learned the hard way so you don't have to.

📋 Prep is everything The more materials you give Claude upfront, the fewer rounds of changes you'll need.
📊 Templates Excel is king Get this file perfect first — with real SharePoint links. Placeholder links = broken buttons on live site.
📦 Bundle change requests 5 changes in one message is 10x better than 5 separate messages.
📤 Re-upload after local edits If you edited files locally, upload them back before requesting more changes from Claude.
📸 Photos: name them simply Use Jill.png not "jill photo final (2).PNG". Crop to square. Any size works — CSS handles the rest.
🚫 Skip SharePoint embed SharePoint strips CSS/JS from embedded HTML. Your beautiful site will look like plain text. Use GitHub Pages instead.
🔍 Test after every batch Open index.html in browser after each batch. Catch issues early before they compound.
✏️ "PwC" not "PWC" Tell Claude explicitly if casing matters. Small brand details get missed if you don't specify.
🤖 Chatbot is fragile A single misplaced quote in chatbot.js breaks the entire chatbot. Always test it after changes.
🔗 Test all links Click every button on every page after deployment. Placeholder URLs from the build will need replacing.
📱 Check mobile view Press F12 in your browser → toggle device toolbar to see how it looks on phones. The site is responsive.
🔄 You can always come back Upload your files to a new Claude chat anytime for more changes. Just provide context on what you need.
